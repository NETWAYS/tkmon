NAME

Alert::Handler::Validation - Validate heartbeats/serials via REST API

VERSION

This document describes Alert::Handler::Validation version 0.0.1

SYNOPSIS

Example: 

	my $retCode = valAuthKey('9WNa8V2P86Q');
	my $retCode = valAuthKey('r2XW84Nfi6v','9000091290');

DESCRIPTION

Alert::Handler::Validation calls two different REST APIs - one for alert heartbeats and one for serial numbers. Both APIs require a valid auth key to return 200 on success. Else 403 as an invalidation notice is returned. In order to check an auth/serial combination a valid serial number must be provided as a second argument. If the monitoring service has not been bought by the customer 402 is returned.

METHODS

valAuthKey

Example: 

	my $retCode = valAuthKey('9WNa8V2P86Q');
	my $retCode = valAuthKey('r2XW84Nfi6v','9000091290');

Calls the REST API with libcurl an returns the status http code. If the second parameter is defined it must be a valid serial number.

Return Codes for auth keys:

403 - Not a valid auth key.
200 - A valid auth key.

Return Codes for auth/serial combinations:

403 - Not a valid auth/serial key.
402 - Payment required, service not valid for auth/serial.

DIAGNOSTICS

Could not perform REST call.
The call to the libcurl library to carry out the request returned an error.

CONFIGURATION AND ENVIRONMENT

Alert::Handler::Validation requires no configuration files or environment variables.
